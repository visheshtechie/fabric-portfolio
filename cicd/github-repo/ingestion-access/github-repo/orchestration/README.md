# Orchestration & Logging

Building production-grade Data Pipelines in Fabric — from simple end-to-end flows to metadata-driven frameworks with parameter passing across pipelines, notebooks, and stored procedures.

## Topics

- **End-to-end pipelines** — connecting ingestion, warehouse loading, and semantic model refresh in a single orchestration
- **Metadata-driven design** — ForEach loops over config tables, dynamic dataflow execution, JSON metadata frameworks with translator objects for dynamic mapping
- **Pipeline expressions** — fluency with arrays, objects, and nested data structures inside pipeline expressions
- **Parent-child orchestration** — parameter round-trips across parent pipeline → child pipeline → notebook → stored procedure
- **Data quality gates** — schema validation, quarantine logic for non-conforming files, and metadata-driven DQ pipelines
- **IoT pipeline** — temperature data ingestion and transformation in a Lakehouse
