# Solution Architecture

High-level Fabric design decisions — capacity sizing, workspace organisation, domain structure, and disaster recovery. Small module by tutorial count, but each piece is an architectural pattern that shapes everything else in a Fabric tenant.

## Topics

- **Capacity Design** — sizing, workload isolation, and capacity-splitting patterns
- **Workspace Design** — environment separation, ownership boundaries, and dependency management across workspaces
- **Domain Design** — organising data by business domain in Fabric, and the governance implications
- **Disaster Recovery** — backup, restore, and continuity planning for production Fabric workloads

These patterns are applied in the [Real-world Projects](../real-world-projects) module — Sprint 1 of both projects covers capacity and workspace design as the foundation for everything that follows.
