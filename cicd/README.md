# CI/CD

The deepest module in the portfolio — covering Fabric's full CI/CD story: Git integration, Deployment Pipelines, item-level CI/CD across all Fabric artefacts, the Fabric REST API, Fabric CLI, and external automation via GitHub Actions.

## Topics

- **Git integration** — repository setup, branches and policies, commit and sync workflows between Fabric workspaces and Azure DevOps
- **Deployment Pipelines** — setup, item pairing, deployment rules, environment promotion (dev → test → prod)
- **Item-level CI/CD** — Data Pipelines, Notebooks, Power BI Projects (PBIP), Data Warehouse, Lakehouse, SQL Database Projects
- **Variable Libraries** — environment-specific configuration management
- **Fabric REST API** — three-part deep dive covering Git operations, Deployment Pipelines, and utility endpoints
- **Fabric CLI** — command-line workflows for power users
- **Local development** — dev environment setup, AI-assisted workflows
- **External automation** — GitHub Actions for CI/CD workflows that go beyond Fabric's built-in tooling
- **Automated build challenge** — end-to-end automated Fabric build using the REST API

> The structure of this very portfolio repository — Fabric committing to Azure DevOps and mirroring to GitHub via a pipeline that rewrites authorship — is itself an applied CI/CD project.
