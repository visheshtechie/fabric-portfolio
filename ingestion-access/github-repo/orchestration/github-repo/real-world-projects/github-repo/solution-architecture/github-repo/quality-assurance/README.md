# Quality Assurance

Production-grade data quality testing in Fabric using Great Expectations (GX Core 1.0) — covering the full validation lifecycle from foundational drills through a multi-part real-world Data Quality System (DQS) project.

## Topics

- **DQ fundamentals** — introduction to data quality concepts and Great Expectations Core 1.0 in a Fabric context
- **Seven dimensions of data quality** — drill-based fluency across schema validation, missing data, uniqueness, integrity, distribution, freshness, and volume checks
- **DQS project** — building a reusable data quality system end-to-end:
  - GX Context preparation
  - Validating incoming files at the ingestion boundary
  - Validating Spark DataFrames mid-pipeline
  - Validating semantic models at the consumption layer
  - Expectation design challenge — applied design exercise

## Skills

`Great Expectations (GX Core 1.0)` `Schema validation` `Missing data checks` `Uniqueness constraints` `Data integrity` `Freshness checks` `Volume checks` `Distribution analysis` `Spark DataFrame validation` `Semantic model validation` `Reusable DQ frameworks`
